REVISÃO: MODELOS OSI E TCP/IP

Modelo OSI

  1. Aplicação - Interface com utilizador e aplicativos
  2. Apresentação - Formatação de dados, encriptação
  3. Sessão - Estabelecimento e gestão de sessões
  4. Transporte - Controlo de fluxo, segmentação
  5. Rede - Encaminhamento, endereçamento lógico
  6. Enlace - Endereçamento físico, acesso ao meio
  7. Física - Transmissão de bits, sinais eléctricos

Modelo TCP/IP

  1. Aplicação - Combina camadas 5, 6 e 7 do OSI
  2. Transporte - TCP, UDP (equivalente à camada 4 do OSI)
  3. Internet - IP (equivalente à camada 3 do OSI)
  4. Acesso à Rede - Combina camadas 1 e 2 do OSI

Importante: As camadas 1 e 2 do modelo OSI correspondem à camada de Acesso à Rede do modelo TCP/IP. Já as camadas 5, 6 e 7 do modelo OSI correspondem à camada de Aplicação do TCP/IP.

INTRODUÇÃO AO PACKET TRACER

O Cisco Packet Tracer é uma ferramenta de simulação de rede que permite aos estudantes experimentar comportamentos de rede e explorar cenários de configuração.

1 Iniciar o Packet Tracer

Abra o programa Cisco Packet Tracer a partir do menu iniciar ou ambiente de trabalho.

2 Interface do Packet Tracer
  • Painel de dispositivos: Localizado à esquerda, contém todos os dispositivos disponíveis
  • Área de trabalho: Espaço central onde a topologia de rede é construída
  • Modos: Tempo real (Realtime) e Simulação (Simulation)
3 Adicionar dispositivos

Clique na categoria de dispositivo desejada (encaminhadores, comutadores, PCs, etc.) e arraste o dispositivo para a área de trabalho.

4 Conectar dispositivos

Clique no ícone de conexão, seleccione o tipo de cabo apropriado e conecte os dispositivos clicando primeiro em um e depois no outro.

Dica: Pode alternar entre o modo de tempo real e o modo de simulação usando o botão localizado no canto inferior direito da interface. O modo de simulação é essencial para visualizar o fluxo de pacotes através das camadas OSI.

PRÁTICA 1: CAMADAS FÍSICA E DE ENLACE

Configuração da Topologia

1 Adicione dois PCs à área de trabalho

Seleccione End Devices > PC-PT e arraste dois para a área de trabalho

2 Conecte os PCs com um cabo directo (straight-through)

Clique no ícone de conexões > Copper Straight-Through > Clique no primeiro PC e depois no segundo

3 Configure endereços IP para os PCs
  • Clique no PC > guia Desktop > IP Configuration
  • Para PC0: IP: 192.168.1.1, Máscara: 255.255.255.0
  • Para PC1: IP: 192.168.1.2, Máscara: 255.255.255.0

Teste da Camada Física

4 Verifique o estado da conexão física

Observe as luzes de link nas interfaces dos PCs. Verde indica conexão estabelecida.

5 Experimente diferentes tipos de cabos

Substitua o cabo directo por um cabo cruzado (crossover) e observe se há diferença.

Teste da Camada de Enlace

6 Visualize os endereços MAC

Clique em cada PC > Command Prompt > Digite ipconfig /all para ver o endereço MAC

PRÁTICA 1: ANÁLISE DE PACOTES (FÍSICA E ENLACE)

Análise dos Pacotes em Modo de Simulação

1 Entre no modo de simulação

Clique no botão "Simulation" no canto inferior direito do ecrã

2 Configure os filtros para visualizar os pacotes relevantes

Clique em "Edit Filters" e seleccione apenas os protocolos que deseja visualizar (ARP, ICMP)

3 Envie um pacote de um PC para outro

Em um dos PCs, abra o Command Prompt e digite ping 192.168.1.2 (ou o IP do outro PC)

4 Observe o processo ARP (Address Resolution Protocol)

Antes que o ping ICMP seja enviado, observe o pacote ARP que é usado para descobrir o endereço MAC do destinatário

  • Clique no pacote ARP durante a simulação
  • Analise o cabeçalho do quadro Ethernet (Camada 2)
  • Observe os endereços MAC de origem e destino
5 Análise do quadro de dados

Após o ARP, observe o pacote ICMP (ping) sendo encapsulado em um quadro Ethernet

  • Note o preâmbulo Ethernet (início do quadro)
  • Observe o campo de verificação de erros (FCS - Frame Check Sequence)

Importante: Nestes testes, estamos a observar principalmente a Camada 1 (sinais e conexões físicas) e a Camada 2 (endereçamento MAC e quadros Ethernet) do modelo OSI.

PRÁTICA 2: CAMADA DE REDE

Configuração da Topologia com Encaminhador (Router)

1 Crie uma nova topologia com um encaminhador e dois PCs
  • Adicione um encaminhador (Routers > 1841)
  • Adicione dois PCs (End Devices > PC-PT)
2 Conecte os dispositivos
  • Use cabos directos (Copper Straight-Through) para conectar cada PC ao encaminhador
  • PC0 ao FastEthernet0/0 do encaminhador
  • PC1 ao FastEthernet0/1 do encaminhador
3 Configure o encaminhador

Clique no encaminhador > guia CLI e digite os seguintes comandos:

Router>enable
Router#configure terminal
Router(config)#interface FastEthernet0/0
Router(config-if)#ip address 192.168.1.1 255.255.255.0
Router(config-if)#no shutdown
Router(config-if)#exit
Router(config)#interface FastEthernet0/1
Router(config-if)#ip address 192.168.2.1 255.255.255.0
Router(config-if)#no shutdown
Router(config-if)#exit
Router(config)#exit
Router#write memory
4 Configure os PCs em subredes diferentes
  • PC0: IP: 192.168.1.10, Máscara: 255.255.255.0, Gateway: 192.168.1.1
  • PC1: IP: 192.168.2.10, Máscara: 255.255.255.0, Gateway: 192.168.2.1

PRÁTICA 2: TESTE DA CAMADA DE REDE

Testando a Comunicação Entre Subredes

1 Verifique a conectividade local

Em cada PC, faça ping para o seu respectivo gateway:

  • Em PC0: ping 192.168.1.1
  • Em PC1: ping 192.168.2.1
2 Teste a comunicação entre subredes

Em PC0, faça ping para PC1:

  • ping 192.168.2.10
3 Observe a rota

Execute o comando tracert 192.168.2.10 em PC0 para ver a rota que o pacote percorre

Análise no Modo de Simulação

4 Active o modo de simulação

Clique no botão "Simulation" e configure os filtros para mostrar ICMP e IP

5 Envie um pacote e observe o processo

Faça ping novamente e observe cada etapa do processo:

  • Observe o pacote ICMP sendo encapsulado em um pacote IP
  • Note que o destino do quadro Ethernet é o gateway (encaminhador)
  • Observe o encaminhador recebendo o pacote, desencapsulando, verificando a tabela de encaminhamento e reencapsulando para a nova subrede
  • Veja o novo quadro Ethernet sendo criado com o endereço MAC do PC de destino

Importante: Nesta prática estamos a observar principalmente a Camada 3 (Rede) do modelo OSI, que corresponde à camada Internet do modelo TCP/IP. Observe como o encaminhador toma decisões baseadas nos endereços IP (endereçamento lógico).

PRÁTICA 3: CAMADA DE TRANSPORTE

Configuração de Servidor e Clientes

1 Adicione um servidor à topologia

Seleccione End Devices > Server-PT e adicione-o à rede

2 Conecte o servidor ao encaminhador

Use um cabo directo para conectar o servidor à interface FastEthernet0/1 do encaminhador

3 Configure o endereço IP do servidor

Clique no servidor > guia Config > Interface > FastEthernet > Configure:

  • IP: 192.168.2.20
  • Subnet Mask: 255.255.255.0
  • Default Gateway: 192.168.2.1
4 Configure serviços no servidor

Clique no servidor > guia Services e habilite os seguintes serviços:

  • HTTP: On (porta 80)
  • FTP: On (porta 21)
  • DNS: On (porta 53)
5 Configure o serviço HTTP
  • Na guia Services > HTTP, adicione uma página HTML simples
  • Clique em "index.html" e adicione código HTML básico, como:
<html>
<head>
    <title>Teste da Camada de Transporte</title>
</head>
<body>
    <h1>Página de Teste - Modelo OSI</h1>
    <p>Esta página demonstra a camada de transporte e aplicação</p>
</body>
</html>

PRÁTICA 3: TESTE DA CAMADA DE TRANSPORTE (CONTINUAÇÃO)

4 Analise os cabeçalhos TCP

Clique em um dos pacotes TCP e examine:

  • Porta de origem e porta de destino (80 para HTTP)
  • Número de sequência e número de confirmação
  • Flags TCP (SYN, ACK, FIN, etc.)
  • Tamanho da janela (controlo de fluxo)
  • Checksum (verificação de erros)
5 Observe a transferência de dados

Após o handshake, observe os pacotes HTTP sendo transmitidos:

  • Requisição HTTP GET do cliente
  • Resposta HTTP do servidor com código 200 (OK)
  • Observe como os dados são segmentados em múltiplos pacotes TCP, se a página for grande
6 Teste o serviço FTP

Em PC0, tente aceder ao servidor FTP:

  • Desktop > Command Prompt
  • Digite: ftp 192.168.2.20
  • Use nome de utilizador "cisco" e senha "cisco"

Importante: Esta prática demonstra a Camada 4 (Transporte) do modelo OSI, que corresponde à camada de Transporte do modelo TCP/IP. Observe como TCP e UDP operam de forma diferente: TCP é orientado à conexão (handshake) e garante entrega fiável, enquanto UDP é mais simples e sem conexão.

PRÁTICA 4: CAMADAS SUPERIORES (SESSÃO, APRESENTAÇÃO, APLICAÇÃO)

Configuração para DNS e Serviços de Aplicação

1 Configure o serviço DNS no servidor

No servidor, configure o DNS para associar um nome ao endereço IP:

  • Clique no servidor > guia Services > DNS
  • Adicione um registo de host: Name: "servidor", Address: 192.168.2.20
2 Configure o cliente para usar o servidor DNS

Em PC0, defina o servidor DNS:

  • Desktop > IP Configuration
  • Adicione 192.168.2.20 como servidor DNS
3 Adicione mais conteúdo ao servidor web

No servidor, adicione outra página HTML com conteúdo mais complexo:

  • Services > HTTP
  • Adicione uma nova página, ex: "formatacao.html"
  • Adicione conteúdo com diferentes formatações (tabelas, imagens, etc.)
4 Configure um serviço de correio electrónico (SMTP/POP3)

No servidor:

  • Services > EMAIL
  • Crie uma conta: Domain Name: "servidor.local", User Name: "admin"
5 Configure clientes de correio electrónico nos PCs

Em PC0 e PC1:

  • Desktop > Email
  • Configure: Your Name, Email, Server (192.168.2.20)

Nota: As camadas superiores (5-Sessão, 6-Apresentação e 7-Aplicação) do modelo OSI são combinadas na camada de Aplicação do modelo TCP/IP. Estas camadas lidam com a interacção do utilizador, formatação de dados e gestão de sessões.

PRÁTICA 5: RESOLUÇÃO DE PROBLEMAS COM O MODELO OSI

Simulação de Problemas em Cada Camada

1 Problemas na Camada Física (1)

Simule um problema físico:

  • Desconecte um cabo entre dispositivos
  • Tente fazer ping e observe o erro
  • Diagnóstico: Verifique as luzes do link e a integridade da conexão física
2 Problemas na Camada de Enlace (2)

Simule um problema de endereçamento MAC:

  • Adicione um comutador (switch) entre dispositivos
  • Configure um filtro MAC no comutador (MAC Filter)
  • Diagnóstico: Verifique a tabela ARP (arp -a) e a tabela MAC do comutador
3 Problemas na Camada de Rede (3)

Simule um problema de encaminhamento:

  • Configure um endereço IP incorrecto ou máscara de sub-rede errada
  • Desactive uma interface do encaminhador
  • Diagnóstico: Verifique a configuração IP (ipconfig) e a tabela de rotas (route print)
4 Problemas na Camada de Transporte (4)

Simule um problema de porta:

  • Configure uma ACL no encaminhador para bloquear a porta 80 (HTTP)
  • Tente aceder ao servidor web
  • Diagnóstico: Verifique se outros serviços funcionam (telnet 192.168.2.20 80)
5 Problemas nas Camadas Superiores (5-7)

Simule problemas de aplicação:

  • Desactive o serviço HTTP no servidor
  • Configure um nome de domínio incorrecto no DNS
  • Diagnóstico: Teste DNS (nslookup) e conexão directa por IP

COMPARAÇÃO DETALHADA: MODELO OSI vs TCP/IP

Atributo Modelo OSI Modelo TCP/IP
Número de Camadas 7 camadas 4 camadas
Origem Desenvolvido pela ISO como modelo teórico Desenvolvido pelo DoD baseado em protocolos reais
Abordagem Separa as funções de rede em camadas mais específicas Combina funções em camadas mais abrangentes
Adopção Principalmente usado como referência conceptual Amplamente implementado na prática
Protocolos Define serviços, interfaces e protocolos Não distingue claramente entre serviços, interfaces e protocolos
Flexibilidade Rígido, com camadas bem definidas Mais flexível, adaptável a diferentes necessidades

Mapeamento entre os modelos

Importante para Resolução de Problemas: Quando um problema de rede ocorre, o modelo OSI ajuda a isolar em qual camada o problema está a acontecer, permitindo uma abordagem metodológica para resolução. Comece sempre verificando a camada física e suba gradualmente.

! Regra prática para resolução de problemas

"Se não pode fazer PING a um host, o problema está nas camadas 1-3. Se pode fazer PING mas não pode aceder a um serviço específico, o problema está nas camadas 4-7."

RESUMO E AVALIAÇÃO

Pontos-chave da aula prática

  • O modelo OSI é uma estrutura conceptual de 7 camadas que padroniza as funções de um sistema de comunicação de rede.
  • O modelo TCP/IP é uma implementação prática com 4 camadas que é a base da internet.
  • Cada camada do modelo OSI encapsula dados e adiciona seu próprio cabeçalho.
  • Packet Tracer é uma ferramenta excelente para visualizar e entender as operações em cada camada.
  • A abordagem de resolução de problemas baseada em camadas permite identificar e resolver problemas de forma sistemática.

Exercícios Práticos para Avaliação

Exercício 1: Criação de Topologia Completa

Crie uma topologia de rede com dois encaminhadores, dois comutadores e quatro PCs, divididos em duas subredes distintas. Configure todos os dispositivos para permitir a comunicação entre todos os hosts e implemente um servidor web em uma das subredes.

Exercício 2: Análise de Pacotes

Usando o modo de simulação, capture e analise pacotes para os seguintes protocolos:

  1. ARP (Camada 2)
  2. ICMP (Camada 3)
  3. TCP handshake (Camada 4)
  4. HTTP (Camada 7)

Para cada pacote, identifique os cabeçalhos relevantes e explique sua função.

Exercício 3: Resolução de Problemas

O seu colega irá introduzir um problema na sua topologia de rede. Use a abordagem sistemática do modelo OSI para identificar e resolver o problema, documentando cada passo do processo de resolução de problemas.
